# 源码目录

在此目录中编写或编辑游戏代码时，请遵循以下标准。

## 引擎版本警告

LLM 的训练数据早于锁定引擎版本。
**在使用任何引擎 API 之前，始终查阅 `docs/engine-reference/`。**
不要猜测知识截止日期之后的 API 签名——先查文档。

## 编码标准

- 所有公共 API 需要文档注释
- 玩法数值必须**数据驱动**（外部配置文件），绝不可硬编码
- 优先使用依赖注入而非单例以确保可测试性
- 每个新系统需要在 `docs/architecture/` 中有对应的 ADR
- 提交必须引用相关的 Story ID 或设计文档

## 文件路由

将引擎专家 Agent 与正在编写的文件类型匹配。
参见 `CLAUDE.md` → 技术偏好 → 引擎专家 → 文件扩展名路由。

不确定时，使用 `CLAUDE.md` 中配置的主要引擎专家。

## 测试

测试存放在 `tests/` 中——不在 `src/` 中。
如果测试框架尚未存在，运行 `/test-setup` 搭建它。
每个玩法系统应有覆盖其公式和边缘情况的单元测试。

## 验证驱动开发

添加玩法系统时先写测试。
对于 UI 更改，通过截图验证。
在标记工作完成之前，将预期输出与实际输出进行比对。
