# 原型代码标准（放宽）

原型是用于验证想法的可抛弃代码。标准有意放宽以最大化迭代速度。目标是学习，而非生产质量。

## 原型中允许的

- 硬编码数值（不需要数据驱动的配置）
- 最少或没有文档注释
- 简单的架构（不需要依赖注入）
- 单例和全局状态
- 复制粘贴的代码（不需要抽象）
- 保留调试输出
- 临时占位的美术和音频资源
- 快速但不优雅的解决方案

## 仍然需要的

- 每个原型存放在自己的子目录中：`prototypes/[名称]/`
- 每个原型必须有 `README.md` 包含：
  - 正在验证什么假设
  - 如何运行原型
  - 当前状态（进行中 / 已结束）
  - 发现（原型结束时更新）
- 生产代码不得引用或导入 `prototypes/` 中的内容
- 原型不得修改 `prototypes/` 之外的文件
- 原型不得被部署或发布

## 原型成功后

如果原型验证了某个概念，功能进入生产阶段：
1. 原型代码不直接迁移——按照生产标准重写
2. 原型的 `README.md` 发现为生产设计文档提供参考
3. 原型目录保留以便参考，但不再扩展

## 清理

已结束的原型应在捕获发现后归档或删除。
绝不要让原型代码通过增量"清理"演变成生产代码。
